## 1. Tổng quan dự án

**Better Me 2026** là một personal daily tracker app dạng mobile-first (max-width 448px), chạy trên web.

## 6. Logic tính điểm

- **`binary`**: `completed` = 100% weight, `failed` = 0%
- **`rating`**: `good`(100%) / `quite_good`(80%) / `average`(50%) / `bad`(0%)
- N/A task → bị loại khỏi tổng weight, các task còn lại tự tăng tỷ trọng
- Công thức: `normalizedWeight = (weight / totalValidWeight) * 100`, `percent = Math.round(earnedRaw)`

## 9. Luồng UX quan trọng

### PIN / Session
```
Mở app/tab mới → PIN screen → nhập 0009 → vào app
Idle 1 phút    → tự thoát về PIN screen
Refresh trang  → vẫn authed (sessionStorage còn)
```

### Chốt sổ / Mở khóa
```
CHỐT SỔ → confirm modal → confirmSubmit():
  1. setShowConfirmModal(false) + setIsSubmitted(true)  [đồng bộ]
  2. syncToCloud(..., 'submitted')                       [async, lưu netCalories]
  3. Auto-submit tất cả ngày draft trước ngày này        [async, batch]

MỞ KHÓA → handleRevise():
  1. setShowConfirmModal(false) + setIsSubmitted(false)  [đồng bộ, ngay lập tức]
  2. setDoc(status:'draft')                              [fire-and-forget, nền]
```

**Modal guard:** `{showConfirmModal && !isSubmitted && ...}` — modal không bao giờ render khi đã submitted.  
**useEffect safety:** `useEffect(() => { if (isSubmitted) setShowConfirmModal(false); }, [isSubmitted])` — reset modal khi submitted từ bất kỳ nguồn nào.

### Sync strategy
- Task status change → sync ngay
- Text fields (journal, meals, reflections) → sync khi `onBlur`
- `onSnapshot` realtime → tự cập nhật từ thiết bị khác

### netCalories
- Lưu vào Firestore mỗi lần `syncToCloud` (kể cả lúc chốt sổ)
- Nếu thiếu (log cũ): `computeNetFromMeals(log.meals)` tính retroactively
- Chart/streak/weekly đều dùng `getLogNet(log)` để xử lý cả hai trường hợp

## 10. Chi tiết kỹ thuật

### Dark mode
- `@custom-variant dark (&:where(.dark, .dark *))` trong `src/index.css`
- Toggle `.dark` trên `document.documentElement`

### PIN auth
- State `isAuthed` từ `sessionStorage.getItem('bm_auth') === '1'`
- Sai → animation shake (`@keyframes shake` trong index.css)
- Đổi pass: sửa `APP_PIN = '0009'` đầu file

### Auto-logout
- `useEffect` + `addEventListener` trên 6 events: `mousemove`, `mousedown`, `touchstart`, `keydown`, `scroll`, `click`
- `clearTimeout` + `setTimeout(60_000)` mỗi lần có event
- Khi timeout: `sessionStorage.removeItem('bm_auth')` + `setIsAuthed(false)`

### Recharts scrollable chart
- `BarChart width={data.length * 36}` fixed trong `div overflow-x-auto`
- `useRef` + `useEffect` auto-scroll về phải (ngày mới nhất) khi mở tab Stats
- Calorie chart `YAxis domain={[0, dataMax => max(dataMax*1.15, budget*1.3)]}` — luôn hiện bar overbudget

### PWA icons
- `scripts/generate-icons.mjs` — pure Node.js, không deps, tạo solid black PNG với circle design
- Run: `node scripts/generate-icons.mjs`

### Auto-submit previous days
- `confirmSubmit` gọi `Promise.all(prevDrafts.map(log => setDoc(..., {status:'submitted', netCalories:...})))`
- Chỉ tác động ngày đã có document trong Firestore (status: 'draft')

```bash
# src/index.css:
# @import "tailwindcss"; @config "../tailwind.config.js";

git init && git add . && git commit -m "init"
git remote add origin <url> && git push -u origin main
npx vercel --prod
```

**Firestore rules:**
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```
